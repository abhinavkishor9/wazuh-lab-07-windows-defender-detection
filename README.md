# wazuh-lab-07-windows-defender-detection
## Overview

This lab demonstrates how Windows Defender Antivirus events can be collected, forwarded, and investigated inside Wazuh.

The objective is to understand how endpoint security events generated by Windows Defender appear within the SIEM and how analysts can validate detections using Windows Event Logs, PowerShell, and the Wazuh Threat Hunting dashboard.

---

## Objectives

- Generate Windows Defender events
- Verify Defender operational logs
- Confirm Wazuh agent collection
- Investigate alerts inside Wazuh
- Correlate endpoint logs with SIEM events
- Practice SOC investigation workflow

---

## MITRE ATT&CK

Although Windows Defender events are defensive telemetry, they commonly relate to attacker execution techniques such as:

- T1059 – Command and Scripting Interpreter
- T1204 – User Execution
- T1562 – Impair Defenses (attempted AV bypass)
- T1105 – Ingress Tool Transfer

---

## Lab Workflow

1. Verify Defender service
2. Generate Defender activity
3. Collect Windows Event Logs
4. Confirm Wazuh ingestion
5. Hunt Defender events
6. Review event metadata
7. Validate timestamps
8. Document investigation

---

## PowerShell Verification

Check Defender events:

```powershell
Get-WinEvent -LogName "Microsoft-Windows-Windows Defender/Operational" -MaxEvents 20
```

Check Wazuh service:

```powershell
Get-Service WazuhSvc
```

Check Sysmon:

```powershell
Get-Service Sysmon64
```
